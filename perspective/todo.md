# Todo 视角

## 定义

**Todo 视角**：用户已经知道要做什么，只需要 Agent 执行。

## 特点

- 用户清楚知道具体目标
- 不需要探索"做什么"，只需要执行"怎么做"
- Query 相对直接，但可能不告诉具体路径

## 典型 Query 模式

```
"请修复数据库连接超时的配置"
"将端口从 X 改为 Y"
"执行构建脚本生成产物"
```

## 与其他视角的区别

| 视角 | 用户知道 | 示例 |
|------|---------|------|
| todo | 知道要做什么 | "修复端口配置" |
| reference | 需要查资料才知道怎么做 | "根据最佳实践配置" |
| explore | 需要调研后才知道做什么 | "系统有问题，请排查" |

## 设计要点

1. **Query 告诉目标，不告诉路径**
   - 可以说"修复端口配置"
   - 不要说"修改 config/database.yaml"

2. **探索距离来自文件结构**
   - 用户知道目标，但需要找到正确的文件
   - 可能有多个类似文件需要区分

3. **答案值仍需从环境获取**
   - 即使目标明确，具体值也不能预测

## 适用场景

- 简单修复任务
- 配置变更
- 脚本执行
