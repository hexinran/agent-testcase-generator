# Reference 视角

## 定义

**Reference 视角**：用户知道要做什么，但需要查阅参考资料才知道具体怎么做。

## 特点

- 用户有一个目标
- 需要参考文档、最佳实践、故障单等资料
- 正确答案藏在参考资料中

## 典型 Query 模式

```
"根据最近的故障单修复配置"
"按照性能测试报告的建议优化参数"
"根据架构迁移文档更新配置"
```

## 与其他视角的区别

| 视角 | 用户知道 | 需要 |
|------|---------|------|
| todo | 知道要做什么和大概怎么做 | 找到正确文件 |
| **reference** | **知道要做什么** | **查资料找具体值** |
| explore | 不确定具体要做什么 | 先调研再决定 |

## 设计要点

1. **必须有参考文档**
   - 故障单、技术文档、最佳实践
   - 正确值必须从参考文档获取

2. **Query 指向参考方向**
   - "根据故障单..."
   - "按照性能报告..."
   - "参考架构文档..."

3. **参考文档不能太直接**
   - 不是"请改为 19847"
   - 而是在上下文中提供值

## 适用场景

- 故障排查任务
- 遵循规范的配置变更
- 参考最佳实践的优化
