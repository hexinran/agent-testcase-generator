# D7：大师级（11-15步）

## 特征

- **超高复杂度**：信息分散在 6+ 个文件
- **极度模糊**：Query 几乎没有技术细节
- **跨子系统**：涉及多个子系统的配置和交互
- **需要系统性分析**：需要全面理解架构

## 参数要求

| 维度 | 要求 |
|------|------|
| Golden Action 步数 | 11-15 步 |
| 环境文件数 | 25-35 个 |
| 信息分散度 | 6+ 个文件 |
| 干扰项数量 | 12+ 个 |

## Query 示例

```
"生产环境核心业务功能出现严重故障，影响大量用户。
监控显示多个指标异常，但根因不明。
请紧急排查所有相关系统配置，找出问题并修复。"
```

## 环境示例

```
25-35 个文件，包括：
- 完整的微服务架构（5+ 个服务）
- 基础设施配置（数据库、缓存、消息队列、网关）
- 多层级的配置文件
- 大量日志和监控数据
- 架构文档、故障案例库
- 大量干扰文件（12+ 个）
```

## 设计要点

- Query 极度模糊，只描述业务影响
- 需要系统性排查多个子系统
- 信息深度隐藏，需要多次推理
- 适合测试 Agent 的全局架构理解能力

## Haiku 预期表现

D7 难度题目，Haiku 大概率失败。这是预期行为，用于区分强弱模型。

## 适用场景

- 架构级问题诊断
- P0 级生产故障排查
- 高级 Agent 能力测试
